# SequenceEmulator

## Overview

SequenceEmulator is a deep learning project designed to predict or emulate atmospheric profiles using a transformer-based neural network implemented in PyTorch. It tackles this as a regression task, learning complex relationships within and between different types of atmospheric sequence data. The model can handle multiple input sequences simultaneously and incorporate global context features.

## Model Architecture (`src/model.py`)

The core model is the `MultiEncoderTransformer`, an encoder-only transformer architecture with the following components:

* **Multi-Encoder Design:** Can process multiple distinct input sequence types (e.g., different kinds of atmospheric measurements or simulations) concurrently. Each sequence type is handled by a dedicated `SequenceEncoder`.
* **`SequenceEncoder`:**
    * Applies a linear projection to map input features to the model dimension (`d_model`).
    * Optionally includes a `ConvBlock1D` module: A stack of 1D dilated convolutions with Group Normalization, GELU activation, Dropout, and residual connections, applied before the main transformer layers. This can help capture local patterns.
    * Adds `SinePositionalEncoding` to inject information about the position within the sequence.
    * Uses a standard PyTorch `TransformerEncoder` stack (composed of `TransformerEncoderLayer` with multi-head self-attention and feed-forward networks) for sequence processing. Supports `norm_first` configuration.
* **Global Feature Handling (Optional):**
    * A `GlobalEncoder` (MLP with GELU, Dropout, LayerNorm) processes non-sequential global input features.
    * `FiLMLayer` (Feature-wise Linear Modulation) uses the encoded global features to generate scaling (gamma) and shifting (beta) parameters, which modulate the output of the sequence encoders, effectively conditioning them on global context.
* **Interaction between Sequences (Optional):**
    * `CrossAttention` module (standard multi-head attention) allows one sequence type to attend to the information encoded in another sequence type, enabling interaction and information sharing between different inputs. This is typically applied after individual sequence encoding and FiLM conditioning.
* **Output Head:** A final linear layer maps the encoded representation of a designated `output_seq_type` to the desired target variable dimension.
* **Factory Function:** `create_prediction_model` builds the model instance from a configuration dictionary.

## Directory Structure

* `src/`: Contains the main Python source code.
    * `main.py`: Main script for running tasks.
    * `model.py`: Model definition.
    * `train.py`: Training loop implementation.
    * `dataset.py`: Data loading and handling.
    * `normalizer.py`: Data normalization logic.
    * `hyperparams.py`: Hyperparameter tuning logic.
    * `utils.py`: Utility functions.
    * `hardware.py`: Hardware/device configuration helpers.
* `data/`: Default root directory for all data artefacts.
    * `profiles/`: **Input:** Place raw atmospheric profile JSON files here.
    * `normalized_profiles/`: **Output:** Normalized profiles generated by the `--normalize` step.
    * `model/`: **Output:** Trained models (`*.pt`), logs (`training_log.csv`), test metrics (`test_metrics.json`), etc.
    * `tuning_results/`: **Output:** Results from hyperparameter tuning runs.
* `inputs/`: Contains configuration files.
    * `model_input_params.jsonc`: **Input:** Main configuration file defining model parameters, variables, training settings, etc. (Supports JSON with comments).
* `jupyter_notebooks/`: Contains notebooks for visualization.
* `testing/`: Contains testing scripts (not fully detailed in this analysis).
* `run_sbatch*.sh`: Example Slurm batch scripts for running on a cluster (like Gattaca).

## Setup

1.  **Clone the repository:**
    ```bash
    git clone <repository_url>
    cd Emulator-2
    ```

2.  **Dependencies:** The project requires Python 3 and several libraries. Key dependencies include:
    * PyTorch (including `torchvision`, `torchaudio` if installed via PyTorch website recommendations)
    * NumPy
    * Pandas (for reading training logs in notebooks)
    * Matplotlib (for plotting in notebooks)
    * Optuna (for hyperparameter tuning)
    * jsoncomment (for loading `.jsonc` config files)

3.  **Environment Setup (Recommended):** Use Conda to create an environment. The provided sbatch scripts use an environment named `nn`.
    ```bash
    # Create a new conda environment (e.g., named 'emulator_env')
    conda create -n emulator_env python=3.10 # Or another Python 3 version
    conda activate emulator_env

    # Install PyTorch (select the command appropriate for your system/CUDA version from pytorch.org)
    # Example for CUDA 11.8:
    # conda install pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia

    # Install other dependencies
    conda install numpy pandas matplotlib optuna
    pip install jsoncomment # Or conda install if available in your channels
    ```

4.  **CUDA Setup (for GPU):** If using a GPU, ensure you have a compatible NVIDIA driver and CUDA toolkit installed. The `run_sbatch_gattaca_gpu.sh` script attempts to load `cuda11.8`. Training efficiency highly benefits from GPU usage.

## Usage / Workflow

The main script `src/main.py` orchestrates the different tasks based on command-line arguments.

1.  **Prepare Data:**
    * Place your raw atmospheric profile data, formatted as individual JSON files (e.g., `prof_001.json`, `prof_002.json`, ...), into the `data/profiles/` directory. Each JSON should contain keys for pressure, temperature, and any other variables used as inputs or targets.

2.  **Configure:**
    * Edit the `inputs/model_input_params.jsonc` file to define:
        * Input, target, and global variables.
        * Sequence types and their corresponding variables.
        * Sequence lengths.
        * Model hyperparameters (`d_model`, `nhead`, `num_encoder_layers`, etc.).
        * Training settings (batch size, learning rate, epochs, optimizer, etc.).
        * Normalization settings.
        * Paths (though defaults usually point within `data/`).

3.  **Normalize Data:**
    * Run the normalization step. This calculates global statistics (mean, std dev, etc.) based on the settings in the config file and saves normalized versions of the profiles.
    ```bash
    python src/main.py --normalize
    ```
    * Normalized files will be saved to `data/normalized_profiles/`. A `normalization_metadata.json` file containing the calculated statistics will also be created there.

4.  **Train Model:**
    * Train the model using the specified configuration and the normalized data:
    ```bash
    python src/main.py --train
    ```
    * Outputs are saved to `data/model/`, including:
        * `best_model.pt`: Checkpoint with the best validation loss.
        * `final_model.pt`: Checkpoint at the end of training.
        * `training_log.csv`: Epoch-wise training and validation loss, learning rate, and time.
        * `test_metrics.json`: Loss on the held-out test set.
        * `test_set_info.json`: Indices used for the test split.

5.  **Hyperparameter Tuning (Optional):**
    * Run Optuna-based hyperparameter search. The search space should be defined within the `model_input_params.jsonc` under relevant keys (often structured for Optuna).
    ```bash
    python src/main.py --tune
    ```
    * Results, including best parameters found and study details, are typically saved in `data/tuning_results/`.

## Visualization

The `jupyter_notebooks/` directory contains notebooks to help visualize the data and results:
* `plot_atm.ipynb`: Plots sample Pressure-Temperature profiles from the raw data.
* `plot_model_training.ipynb`: Plots the training and validation loss curves using the `training_log.csv` file.

## Key Scripts

* **`src/main.py`**: Main entry point. Parses arguments and calls relevant functions for normalization, training, or tuning.
* **`src/model.py`**: Defines the `MultiEncoderTransformer` PyTorch model architecture.
* **`src/train.py`**: Implements the `ModelTrainer` class responsible for the training, validation, and testing loops, checkpointing, and logging.
* **`src/dataset.py`**: Defines the `AtmosphericDataset` for loading data from normalized JSON files and a custom collate function.
* **`src/normalizer.py`**: Contains the `DataNormalizer` class for calculating statistics and applying normalization.
* **`src/hyperparams.py`**: Contains functions related to setting up and running the Optuna hyperparameter search.
* **`src/utils.py`**: General utility functions (config loading, logging setup, random seeding).
* **`src/hardware.py`**: Helpers for CUDA/CPU device selection and DataLoader settings.
